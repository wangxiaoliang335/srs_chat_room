# 语音翻译服务 - 文件说明

## 已创建的文件

### 核心服务文件

1. **`audio_translation_service.py`** - 主翻译服务
   - 从SRS接收音频流
   - 调用百度ASR识别语音
   - 调用百度MT翻译文本
   - 调用百度TTS合成语音
   - 推送翻译后的音频到SRS

2. **`baidu_asr_client.py`** - 百度语音识别客户端
   - 实现百度ASR API调用
   - 将音频转换为文本

3. **`baidu_mt_client.py`** - 百度机器翻译客户端（在`baidu_asr_client.py`中）
   - 实现百度MT API调用
   - 将中文文本翻译为英文

4. **`tts_service.py`** - 文本转语音服务
   - 实现百度TTS API调用
   - 将英文文本转换为语音

5. **`callback_server.py`** - HTTP回调服务器
   - 接收SRS的HTTP Hooks回调
   - 自动启动/停止翻译服务
   - 管理翻译服务进程

### 配置文件

6. **`audio_translation_config.json`** - 配置文件模板
   - 百度API配置
   - SRS服务器配置
   - 音频参数配置

7. **`conf/rtc_with_translation.conf`** - SRS配置文件
   - 包含HTTP Hooks配置
   - 自动调用翻译服务

### 文档文件

8. **`README_语音翻译.md`** - 详细技术文档
   - 功能说明
   - 架构设计
   - API说明
   - 扩展功能

9. **`使用说明.md`** - 快速使用指南
   - 安装步骤
   - 配置方法
   - 使用流程
   - 故障排查

10. **`start_translation_service.sh`** - 启动脚本
    - 自动检查环境
    - 安装依赖
    - 启动服务

### 依赖文件

11. **`requirements.txt`** - Python依赖
    - requests
    - flask

## 工作流程

```
用户说话（中文）
    ↓
SRS接收音频流
    ↓
翻译服务从SRS拉取音频（HTTP-FLV）
    ↓
FFmpeg转换音频格式（AAC → PCM）
    ↓
百度ASR识别（PCM → 中文文本）
    ↓
百度MT翻译（中文文本 → 英文文本）
    ↓
百度TTS合成（英文文本 → 英文语音PCM）
    ↓
FFmpeg编码音频（PCM → AAC）
    ↓
推送到SRS（RTMP）
    ↓
其他用户接收翻译后的英文语音
```

## 快速开始

### 1. 安装依赖

```bash
pip3 install -r requirements.txt
```

### 2. 配置环境变量

```bash
export BAIDU_API_KEY="your_api_key"
export BAIDU_SECRET_KEY="your_secret_key"
export SRS_URL="http://localhost:8080"
```

### 3. 启动服务

```bash
./start_translation_service.sh
```

### 4. 配置SRS

使用提供的配置文件启动SRS：

```bash
./objs/srs -c conf/rtc_with_translation.conf
```

## 文件依赖关系

```
callback_server.py
    └── 启动和管理
        └── audio_translation_service.py
                ├── baidu_asr_client.py (ASR识别)
                ├── baidu_mt_client (在asr_client中，MT翻译)
                └── tts_service.py (TTS合成)
```

## 注意事项

1. **API密钥**：需要在百度AI开放平台申请三个服务
   - 语音识别（ASR）
   - 机器翻译（MT）
   - 语音合成（TTS）

2. **FFmpeg**：必须安装FFmpeg用于音频处理

3. **网络**：需要能访问百度API服务器

4. **延迟**：翻译过程会有1-3秒延迟

5. **资源**：每个房间需要2个FFmpeg进程
