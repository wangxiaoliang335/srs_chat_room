# 语音翻译服务 - 快速使用指南

## 一、准备工作

### 1. 安装依赖

```bash
# 安装Python依赖
pip3 install -r requirements.txt

# 确保已安装FFmpeg
ffmpeg -version
```

### 2. 获取百度API密钥

1. 访问 [百度AI开放平台](https://ai.baidu.com/)
2. 注册/登录账号
3. 创建应用，获取以下服务：
   - **语音识别（ASR）**：用于将语音转为文本
   - **机器翻译（MT）**：用于将文本翻译为英文
   - **语音合成（TTS）**：用于将英文文本转为语音
4. 获取 `API Key` 和 `Secret Key`

> **费用说明**：这三个服务都是付费的，但都有免费额度供测试使用：
> - **ASR（语音识别）**：按需付费约 0.0015元/分钟，有免费测试额度
> - **MT（机器翻译）**：按量付费约 0.028元/次，有免费测试额度（如1万次）
> - **TTS（语音合成）**：类似定价模式，有免费测试额度
> - 免费额度通常有效期12个月，超出后按实际使用量计费
> - 建议先使用免费额度测试，确认功能后再考虑付费

### 3. 配置环境变量

```bash
export BAIDU_API_KEY="你的API_Key"
export BAIDU_SECRET_KEY="你的Secret_Key"
export SRS_URL="http://localhost:8080"  # SRS服务器地址
```

## 二、启动服务

### 方式1：使用启动脚本（推荐）

```bash
./start_translation_service.sh
```

### 方式2：手动启动

```bash
# 1. 启动回调服务器
python3 callback_server.py &

# 2. 启动翻译服务（为特定房间）
ROOM_ID=room_123 python3 audio_translation_service.py
```

## 三、配置SRS

### 1. 使用提供的配置文件

```bash
# 使用支持翻译的配置文件启动SRS
./objs/srs -c conf/rtc_with_translation.conf
```

### 2. 或手动修改SRS配置

在SRS配置文件中添加HTTP Hooks：

```conf
vhost __defaultVhost__ {
    http_hooks {
        enabled         on;
        on_publish      http://127.0.0.1:8085/api/v1/streams/on_publish;
        on_unpublish    http://127.0.0.1:8085/api/v1/streams/on_unpublish;
    }
}
```

## 四、使用流程

### 1. 用户发布音频

用户A通过WebRTC发布音频到房间 `room_123`：

```javascript
// 客户端代码示例
const pc = new RTCPeerConnection();
// ... 配置WebRTC
// 发布到: webrtc://your-domain/live/room_123
```

### 2. 自动启动翻译

- SRS检测到有用户发布流
- 调用回调服务器 `/api/v1/streams/on_publish`
- 回调服务器自动启动翻译服务
- 翻译服务开始处理音频流

### 3. 用户接收翻译后的音频

其他用户订阅翻译后的流：

```javascript
// 订阅翻译后的流
const streamUrl = "webrtc://your-domain/live/room_123_translated";
// ... 建立WebRTC连接接收音频
```

## 五、验证服务

### 1. 检查回调服务器

```bash
curl http://localhost:8085/health
# 应该返回: {"status":"ok"}

curl http://localhost:8085/api/v1/streams/status
# 查看当前正在处理的房间列表
```

### 2. 检查翻译服务日志

查看翻译服务的输出日志，确认：
- 成功连接百度API
- 音频流正常接收
- 翻译结果正常返回
- 音频推送成功

### 3. 测试翻译效果

1. 在房间中说话（中文）
2. 订阅翻译后的流 `{room_id}_translated`
3. 应该听到英文翻译

## 六、常见问题

### Q1: 翻译服务无法启动

**检查项**：
- 环境变量是否正确设置
- Python依赖是否安装完整
- FFmpeg是否已安装
- 网络连接是否正常

### Q2: 无法连接到百度API

**检查项**：
- API密钥是否正确
- 网络是否能访问百度服务器
- API服务是否已开通
- API配额是否充足

### Q3: 翻译延迟较大

**原因**：
- 网络延迟
- API处理时间
- 音频缓冲

**优化**：
- 使用更快的网络
- 减少音频缓冲时间
- 考虑使用流式翻译

### Q4: 翻译后的音频无法播放

**检查项**：
- SRS是否正常运行
- RTMP推流是否成功
- 流名称是否正确（应该是 `{room_id}_translated`）
- 客户端是否正确订阅流

## 七、监控和维护

### 1. 查看服务状态

```bash
# 查看回调服务器状态
curl http://localhost:8085/api/v1/streams/status

# 查看进程
ps aux | grep python
```

### 2. 查看日志

翻译服务的日志会输出到控制台，包括：
- 音频接收状态
- API调用结果
- 错误信息

### 3. 停止服务

```bash
# 停止所有翻译服务
pkill -f audio_translation_service.py

# 停止回调服务器
pkill -f callback_server.py
```

## 八、性能优化

1. **多房间处理**：服务支持同时处理多个房间
2. **资源监控**：注意CPU和内存使用情况
3. **API配额**：监控百度API的使用量
4. **网络优化**：使用CDN或优化网络路径

## 九、安全建议

1. **API密钥安全**：
   - 不要将密钥提交到代码仓库
   - 使用环境变量或密钥管理服务

2. **网络安全**：
   - 使用HTTPS连接
   - 限制回调服务器的访问IP

3. **访问控制**：
   - 限制谁可以发布流
   - 限制翻译服务的启动权限

4. 网络连接情况
