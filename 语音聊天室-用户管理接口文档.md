# 语音聊天室 - 用户管理接口文档

本文档记录了语音聊天室业务服务器（`callback_server.py`）提供的用户管理和权限控制接口。

> **基础信息**
> - **默认端口**: 8085 (HTTP API + WebSocket/Socket.IO)
> - **协议**: HTTP/HTTPS
> - **响应格式**: JSON
> - **编码**: UTF-8

---

## 一、用户角色与状态

### 1.1 用户角色

| 角色 | 值 | 说明 | 权限 |
|------|------|------|------|
| 群主 | `owner` | 房间创建者 | 最高权限，可管理所有成员 |
| 管理员 | `admin` | 管理员 | 可禁言、禁麦、踢人 |
| 普通成员 | `member` | 普通成员 | 基本发言权限 |
| 访客 | `guest` | 访客 | 受限权限 |

### 1.2 用户状态

| 状态 | 值 | 说明 |
|------|------|------|
| 正常 | `normal` | 可以正常发言 |
| 禁言 | `muted` | 不能发言 |
| 禁麦 | `mic_off` | 不能使用麦克风发布 |

## 二、房间管理接口

### 2.1 创建房间
- **路径**: `/api/v1/room`
- **方法**: POST
- **描述**: 创建一个新房间，创建者自动成为群主

**请求体**:
```json
{
    "room_id": "room_123",
    "owner_id": "user_001",
    "name": "测试房间"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| room_id | string | 是 | 房间ID（唯一标识） |
| owner_id | string | 是 | 群主用户ID |
| name | string | 否 | 房间名称（默认等于room_id） |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "room_id": "room_123",
        "name": "测试房间",
        "owner_id": "user_001",
        "created_at": "2024-01-01 10:00:00",
        "max_members": 100,
        "allow_speak": true,
        "member_count": 1
    }
}
```

---

### 2.2 获取房间信息
- **路径**: `/api/v1/room/{room_id}`
- **方法**: GET
- **描述**: 获取指定房间的详细信息

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "room_id": "room_123",
        "name": "测试房间",
        "owner_id": "user_001",
        "created_at": "2024-01-01 10:00:00",
        "max_members": 100,
        "allow_speak": true,
        "member_count": 5
    }
}
```

---

### 2.3 删除房间
- **路径**: `/api/v1/room/{room_id}`
- **方法**: DELETE
- **描述**: 删除指定房间（仅群主可操作）

**查询参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| operator_id | string | 是 | 操作者ID（必须是群主） |

**响应示例**:
```json
{
    "code": 0,
    "message": "success"
}
```

- **方法**: PUT

**请求体**:

- **路径**: `/api/v1/room/{room_id}/member/{user_id}/kick`

- **描述**: 将成员踢出房间

**查询参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| operator_id | string | 是 | 操作者ID（群主或管理员） |

- 群主可以踢任何人
- 管理员可以踢普通成员
- 普通成员不能踢人

**响应示例**:
```json

    "code": 0,

}
```

## 四、敲门接口

### 4.1 敲门请求
- **路径**: `/api/v1/room/{room_id}/knock`
- **方法**: POST
- **描述**: 访客请求加入房间，通知房主

**请求体**:
```json
{
    "user_id": "visitor_001",
    "message": "你好，我想加入聊天"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user_id | string | 是 | 敲门者用户ID |
| message | string | 否 | 留言信息 |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "room_id": "room_123",
        "owner_id": "user_001",
        "knocker_id": "visitor_001"
    }
}
```

**错误码**:
| code | 说明 |
|------|------|
| 404 | 房间不存在 |
| 400 | 用户已在房间中 |

---

### 4.2 接受敲门
- **路径**: `/api/v1/room/{room_id}/knock/accept`
- **方法**: POST
- **描述**: 房主或管理员接受访客的敲门请求，自动将访客加入房间

**请求体**:
```json
{
    "operator_id": "user_001",
    "knocker_id": "visitor_001",
    "role": "member"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| operator_id | string | 是 | 操作者ID（房主或管理员） |
| knocker_id | string | 是 | 被接受的敲门者ID |
| role | string | 否 | 分配的角色（默认member） |

**权限**: 仅房主或管理员可操作

**响应示例**:
```json
{
    "code": 0,
    "message": "success"
}
```

**事件通知**:
- 敲门者收到 `room_knock_accepted` 事件
- 房间内其他成员收到 `member_joined` 事件

---

### 4.3 拒绝敲门
- **路径**: `/api/v1/room/{room_id}/knock/reject`
- **方法**: POST
- **描述**: 房主或管理员拒绝访客的敲门请求

**请求体**:
```json
{
    "operator_id": "user_001",
    "knocker_id": "visitor_001",
    "reason": "房间已满"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| operator_id | string | 是 | 操作者ID（房主或管理员） |
| knocker_id | string | 是 | 被拒绝的敲门者ID |
| reason | string | 否 | 拒绝原因 |

**权限**: 仅房主或管理员可操作

**响应示例**:
```json
{
    "code": 0,
    "message": "success"
}
```

**事件通知**:
- 敲门者收到 `room_knock_rejected` 事件

---

### 4.4 敲门流程说明

```
┌─────────────┐                              ┌─────────────┐
│   访客      │                              │    房主     │
└──────┬──────┘                              └──────┬──────┘
       │                                            │
       │  1. POST /knock (请求加入)                  │
       │───────────────────────────────────────────►│
       │                                            │
       │              2. room_knock 事件            │
       │               (Socket.IO私人通知)           │
       │◄───────────────────────────────────────────│
       │                                            │
       │  ┌─────────────────────────────────────┐   │
       │  │  房主看到敲门通知，可以选择：         │   │
       │  │  • 接受 → POST /knock/accept        │   │
       │  │  • 拒绝 → POST /knock/reject        │   │
       │  └─────────────────────────────────────┘   │
       │                                            │
       │  3. POST /knock/accept (接受)              │
       │     或 POST /knock/reject (拒绝)            │
       │◄───────────────────────────────────────────│
       │                                            │
       │  4. room_knock_accepted/rejected 事件      │
       │   (Socket.IO私人通知)                       │
       │───────────────────────────────────────────►│
       │                                            │
```

**客户端订阅说明**:
- 房主订阅房间时会自动订阅私人通知房间 `user_{owner_id}`
- 敲门通知会发送到该私人房间
- 敲门者也会自动订阅自己的私人通知房间

## 七、权限规则汇总

| 操作 | 群主 | 管理员 | 普通成员 |
|------|:----:|:------:|:--------:|
| 禁言/禁麦成员 | ✅ | ✅ | ❌ |
| 解除禁言/禁麦 | ✅ | ✅ | ❌ |
| 全体禁言 | ✅ | ❌ | ❌ |
| 踢人 | ✅ | ✅ (仅成员) | ❌ |
| 修改成员角色 | ✅ | ❌ | ❌ |
| 删除房间 | ✅ | ❌ | ❌ |

**重要限制**:
- 群主不能被禁言、禁麦或踢出
- 群主的角色不可被修改
- 管理员不能踢群主或其他管理员

## 八、WebSocket 实时通知

客户端可以通过 WebSocket 连接接收房间事件的实时推送通知。

### 7.1 连接方式

**Socket.IO 连接地址**: `http://localhost:8085` (推荐，支持自动重连)

**WebSocket 连接地址**: `ws://localhost:8085/ws`

| 连接方式 | 地址 | 说明 |
|---------|------|------|
| Socket.IO | `http://localhost:8085` | 推荐，支持 WebSocket 和 HTTP 长轮询 |
| WebSocket | `ws://localhost:8085/ws` | 原生 WebSocket 连接 |

**WebSocket URL 参数** (可选，通过 query 传递):
| 参数 | 说明 |
|------|------|
| room | 房间ID |
| user | 用户ID |

**完整示例**:
```
ws://localhost:8085/ws?room=room_123&user=user_001
```

### 7.2 订阅消息

连接建立后，发送订阅消息：

```json
{
    "type": "subscribe",
    "room_id": "room_123",
    "user_id": "user_001"
}
```

**响应**:
```json
{
    "type": "subscribed",
    "room_id": "room_123",
    "user_id": "user_001"
}
```

### 7.3 取消订阅

```json
{
    "type": "unsubscribe",
    "room_id": "room_123"
}
```

### 7.4 心跳检测

```json
{"type": "ping"}
```

**服务端响应**:
```json
{"type": "pong"}
```

### 7.5 事件类型列表

| 事件类型 | 说明 | 触发时机 |
|----------|------|----------|
| `member_joined` | 成员加入 | 用户加入房间时 |
| `member_left` | 成员离开 | 用户主动离开房间时 |
| `member_kicked` | 成员被踢 | 用户被踢出房间时 |
| `member_role_changed` | 成员角色变更 | 用户角色被修改时 |
| `member_muted` | 成员被禁言 | 用户被禁言时 |
| `member_unmuted` | 成员被解除禁言 | 用户被解除禁言时 |
| `member_mic_disabled` | 成员被禁麦 | 用户麦克风被禁用时 |
| `member_mic_enabled` | 成员被解除禁麦 | 用户麦克风被启用时 |
| `room_muted_all` | 全体禁言开启 | 群主开启全体禁言时 |
| `room_unmuted_all` | 全体禁言关闭 | 群主关闭全体禁言时 |
| `room_created` | 房间创建 | 新房间创建时 |
| `room_deleted` | 房间删除 | 房间被删除时 |
| `room_knock` | 有人敲门 | 访客请求加入房间时（仅房主收到） |
| `room_knock_accepted` | 敲门被接受 | 敲门者收到被批准加入的消息 |
| `room_knock_rejected` | 敲门被拒绝 | 敲门者收到被拒绝加入的消息 |

### 7.6 事件消息格式

所有事件消息都遵循以下格式：

```json
{
    "event_id": "a1b2c3d4",
    "event_type": "member_muted",
    "room_id": "room_123",
    "user_id": "user_001",
    "operator_id": "owner_001",
    "target_user_id": "",
    "data": {},
    "timestamp": "2024-01-01 10:30:00"
}
```

| 字段 | 说明 |
|------|------|
| event_id | 事件唯一ID |
| event_type | 事件类型 |
| room_id | 房间ID |
| user_id | 触发事件的用户ID |
| operator_id | 操作者ID（如禁言操作者） |
| target_user_id | 目标用户ID |
| data | 额外数据（根据事件类型不同） |
| timestamp | 事件发生时间 |

### 7.7 各事件 data 字段说明

| 事件类型 | data 内容 |
|----------|-----------|
| `member_joined` | 用户完整信息 `{user_id, role, status, ...}` |
| `member_role_changed` | `{old_role, new_role}` |

```javascript
const BASE_URL = 'ws://localhost:8085';
const ROOM_ID = 'room_123';
const USER_ID = 'user_' + Math.random().toString(36).substr(2, 6);

// 1. 连接 WebSocket
const ws = new WebSocket(`${BASE_URL}/ws?room=${ROOM_ID}&user=${USER_ID}`);

// 2. 连接成功
ws.onopen = () => {
    console.log('WebSocket 连接成功');
    
    // 订阅房间
    ws.send(JSON.stringify({
        type: 'subscribe',
        room_id: ROOM_ID,
        user_id: USER_ID
    }));
};

// 3. 接收消息
ws.onmessage = (event) => {
    const message = JSON.parse(event.data);
    console.log('收到消息:', message);
    
    // 根据消息类型处理
    switch (message.type) {
        case 'connected':
            console.log('已连接:', message.message);
            break;
            
        case 'subscribed':
            console.log('已订阅房间:', message.room_id);
            break;
            
        case 'member_joined':
            console.log(`👤 ${message.user_id} 加入了房间 ${message.room_id}`);
            break;
            
        case 'member_left':
            console.log(`👋 ${message.user_id} 离开了房间`);
            break;
            
        case 'member_muted':
            console.log(`🔇 ${message.user_id} 被禁言`);
            break;
            
        case 'room_muted_all':
            console.log(`📢 全体禁言已开启`);
            break;
            
        default:
            console.log('收到其他消息:', message);
    }
};

// 4. 连接关闭
ws.onclose = () => {
    console.log('WebSocket 连接关闭');
};

// 5. 错误处理
ws.onerror = (error) => {
    console.error('WebSocket 错误:', error);
};

// 6. 心跳保活
setInterval(() => {
    if (ws.readyState === WebSocket.OPEN) {
        ws.send(JSON.stringify({ type: 'ping' }));
    }
}, 30000);

// 7. 离开页面时断开连接
window.addEventListener('beforeunload', () => {
    ws.close();
});
```

### 9.4 Python 客户端 (原生 WebSocket)

```python
import websocket
import threading
import json
import time

BASE_URL = "ws://localhost:8085"
ROOM_ID = "room_123"
USER_ID = "user_001"


def on_message(ws, message):
    """收到消息"""
    data = json.loads(message)
    print(f"收到消息: {data}")
    
    msg_type = data.get('type', '')
    
    if msg_type == 'connected':
        print(f"✅ 已连接: {data.get('message', '')}")
    elif msg_type == 'subscribed':
        print(f"📡 已订阅房间: {data.get('room_id', '')}")
    elif msg_type == 'member_joined':
        print(f"👤 {data.get('user_id', '')} 加入了房间")
    elif msg_type == 'member_left':
        print(f"👋 {data.get('user_id', '')} 离开了房间")
    elif msg_type == 'member_muted':
        print(f"🔇 {data.get('user_id', '')} 被禁言")
    elif msg_type == 'room_muted_all':
        print(f"📢 全体禁言已开启")
    elif msg_type == 'pong':
        pass  # 心跳响应


def on_error(ws, error):
    """错误处理"""
    print(f"错误: {error}")


def on_close(ws, close_status_code, close_msg):
    """连接关闭"""
    print(f"连接关闭: {close_status_code} - {close_msg}")


def on_open(ws):
    """连接打开"""
    print("WebSocket 连接成功")
    
    # 订阅房间
    ws.send(json.dumps({
        'type': 'subscribe',
        'room_id': ROOM_ID,
        'user_id': USER_ID
    }))


def heartbeat(ws):
    """心跳保活"""
    while True:
        time.sleep(30)
        try:
            ws.send(json.dumps({'type': 'ping'}))
        except:
            break


if __name__ == '__main__':
    # 创建 WebSocket 连接
    ws_url = f"{BASE_URL}/ws?room={ROOM_ID}&user={USER_ID}"
    ws = websocket.WebSocketApp(
        ws_url,
        on_message=on_message,
        on_error=on_error,
        on_close=on_close,
        on_open=on_open
    )
    
    # 启动心跳线程
    heartbeat_thread = threading.Thread(target=heartbeat, args=(ws,))
    heartbeat_thread.daemon = True
    heartbeat_thread.start()
    
    # 运行 WebSocket
    ws.run_forever()
```
