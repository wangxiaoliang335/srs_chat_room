# 语音聊天室 - SRS对外接口文档

本文档列出了**语音聊天室项目**中，其他业务模块需要调用的SRS HTTP API接口。

> **说明**: 不是所有SRS接口都需要对外暴露，本文档仅包含语音聊天室业务场景中实际需要的接口。

---

## 基础信息

- **默认端口**: 1985
- **协议**: HTTP/HTTPS
- **响应格式**: JSON (除非特别说明)
- **编码**: UTF-8

---

## 一、核心业务接口（必须）

### 1. RTC播放 - 用户加入房间收听
- **路径**: `/rtc/v1/play/`
- **方法**: POST
- **业务场景**: 用户加入语音聊天室，开始接收其他用户的语音
- **请求体**:
```json
{
  "sdp": "offer...",                    // WebRTC SDP offer
  "streamurl": "webrtc://domain/app/room_123",  // 房间流地址
  "api": "http://your-api-server/api",  // 业务服务器回调地址（可选）
  "clientip": "192.168.1.100",          // 客户端IP（可选，会自动获取）
  "tid": "transaction_id"               // 事务ID（可选）
}
```
- **查询参数**:
  - `eip` 或 `candidate`: 服务器公网IP（如果客户端在NAT后）
  - `codec`: 音频编解码器（如：opus）
  - `encrypt`: 是否启用SRTP加密（true/false）
  - `dtls`: 是否启用DTLS（true/false）
- **响应示例**:
```json
{
  "code": 0,
  "server": "server_id",
  "service": "service_id",
  "pid": "process_id",
  "sdp": "answer...",      // WebRTC SDP answer
  "sessionid": "session_xxx" // 会话ID，用于后续管理
}
```

### 2. RTC发布 - 用户开始发言
- **路径**: `/rtc/v1/publish/`
- **方法**: POST
- **业务场景**: 用户开始发言，向房间内其他用户推送自己的语音
- **请求体**:
```json
{
  "sdp": "offer...",
  "streamurl": "webrtc://domain/app/room_123",
  "api": "http://your-api-server/api",
  "clientip": "192.168.1.100",
  "tid": "transaction_id"
}
```
- **查询参数**: 同RTC播放接口
- **响应示例**: 同RTC播放接口

### 3. WHIP协议 - 发布流（标准协议）
- **路径**: `/rtc/v1/whip/`
- **方法**: POST, DELETE
- **业务场景**: 
  - POST: 使用标准WHIP协议发布音频流
  - DELETE: 停止发布（用户停止发言）
- **请求体**: SDP格式（非JSON，纯文本）
- **查询参数**:
  - `app`: 应用名，默认"live"
  - `stream`: 流名（房间ID），如"room_123"
  - `eip`: 服务器公网IP
  - `codec`: 编解码器
- **DELETE参数**:
  - `session`: 会话ID
  - `token`: 会话令牌
- **响应**: 
  - POST: 返回SDP格式，HTTP 201
  - DELETE: HTTP 200

### 4. WHEP协议 - 播放流（标准协议）
- **路径**: `/rtc/v1/whip-play/` 或 `/rtc/v1/whep/`
- **方法**: POST, DELETE
- **业务场景**: 使用标准WHEP协议接收音频流
- **请求体**: SDP格式（非JSON）
- **查询参数**: 同WHIP协议
- **响应**: 同WHIP协议

## 二、房间管理接口（推荐）

### 5. 查询房间流信息
- **路径**: `/api/v1/streams/` 或 `/api/v1/streams/{stream_id}`
- **方法**: GET
- **业务场景**: 
  - 查询所有房间的流信息
  - 查询指定房间的详细信息（有多少人在线、谁在发言等）
- **查询参数**:
  - `stream_id` (路径参数): 流ID（房间ID）
  - `start`: 分页起始位置，默认0
  - `count`: 返回数量，默认10
- **响应示例**:
```json
{
  "code": 0,
  "server": "...",
  "service": "...",
  "pid": "...",
  "streams": [
    {
      "id": "stream_id",
      "vhost": "vhost_name",
      "app": "app_name",
      "name": "room_123",
      "clients": 5,           // 当前在线人数
      "publish": {
        "cid": "client_id",
        "active": true        // 是否有人在发言
      }
    }
  ]
}
```

### 6. 查询房间内用户列表
- **路径**: `/api/v1/clients/` 或 `/api/v1/clients/{client_id}`
- **方法**: GET
- **业务场景**: 
  - 查询房间内所有用户信息
  - 查询指定用户的详细信息
- **查询参数**:
  - `client_id` (路径参数): 客户端ID
  - `start`: 分页起始位置，默认0
  - `count`: 返回数量，默认10
- **响应示例**:
```json
{
  "code": 0,
  "server": "...",
  "service": "...",
  "pid": "...",
  "clients": [
    {
      "id": "client_id",
      "vhost": "vhost_name",
      "stream": "room_123",
      "ip": "192.168.1.100",
      "type": "rtc",
      "publish": {...},
      "play": {...}
    }
  ]
}
```

### 7. 踢掉用户（管理员功能）
- **路径**: `/api/v1/clients/{client_id}`
- **方法**: DELETE
- **业务场景**: 管理员踢掉房间内的某个用户
- **响应示例**:
```json
{
  "code": 0,
  "server": "...",
  "service": "...",
  "pid": "..."
}
```

## 三、系统监控接口（可选）

### 8. 系统摘要信息
- **路径**: `/api/v1/summaries`
- **方法**: GET
- **业务场景**: 监控SRS服务器状态（CPU、内存等）
- **响应示例**:
```json
{
  "code": 0,
  "server": "...",
  "service": "...",
  "pid": "...",
  "data": {
    "cpu": {...},
    "mem": {...},
    "load": {...}
  }
}
```

### 9. Prometheus监控指标
- **路径**: `/metrics`
- **方法**: GET
- **业务场景**: 如果使用Prometheus监控系统，可以采集此接口数据
- **响应格式**: text/plain (Prometheus格式)
- **响应示例**:
```
srs_streams 5
srs_clients 10
srs_cpu_percent 15.5
srs_memory 2048
```

## 四、关于房间管理的重要说明

### ⚠️ SRS不提供创建/解散房间接口

**重要**: SRS作为流媒体服务器，**不提供**创建房间、解散房间的接口。这些功能应该由**业务层**实现。

#### 房间创建流程
```
1. 业务服务器在数据库中创建房间记录
2. 当第一个用户发布流时，SRS会自动创建对应的流
3. 房间ID对应SRS的stream名称（如：room_123）
```

#### 房间解散流程
```
1. 业务服务器决定解散房间
2. 调用 DELETE /api/v1/clients/{client_id} 踢掉所有房间内用户
3. 或等待所有用户自然离开（断开WebRTC连接）
4. 当房间内没有用户时，SRS会自动清理对应的流
5. 业务服务器在数据库中删除房间记录
```

#### 强制解散房间（清空所有用户）
```javascript
// 1. 查询房间内所有用户
const clients = await getRoomClients(roomId);

// 2. 逐个踢掉用户
for (const client of clients) {
  await fetch(`http://srs-server:1985/api/v1/clients/${client.id}`, {
    method: 'DELETE'
  });
}

// 3. 业务层删除房间记录
await deleteRoomFromDatabase(roomId);
```

## 接口使用建议

### 1. 核心业务流程

**用户加入房间流程**:
```
1. 业务服务器验证用户权限
2. 调用 /rtc/v1/play/ 或 /rtc/v1/whep/ 创建播放会话
3. 返回SDP answer给客户端
4. 客户端建立WebRTC连接
```

**用户开始发言流程**:
```
1. 业务服务器验证用户权限（是否允许发言）
2. 调用 /rtc/v1/publish/ 或 /rtc/v1/whip/ 创建发布会话
3. 返回SDP answer给客户端
4. 客户端建立WebRTC连接并推送音频
```

**用户离开房间流程**:
```
1. 客户端断开WebRTC连接（自动）
2. 或调用 DELETE /rtc/v1/whip/?session=xxx&token=xxx 主动断开
```

### 2. 房间状态查询

**查询房间信息**:
```
GET /api/v1/streams/room_123
- 获取房间在线人数
- 获取当前是否有人在发言
- 获取房间状态
```

**查询用户列表**:
```
GET /api/v1/clients/?start=0&count=20
- 获取房间内所有用户
- 分页查询
```

### 3. 管理员功能

**踢掉用户**:
```
DELETE /api/v1/clients/{client_id}
- 管理员操作
- 需要业务服务器验证管理员权限
```

### 4. 房间生命周期管理

**创建房间**（业务层实现）:
```
1. 业务服务器在数据库中创建房间记录
2. 房间ID作为stream名称（如：room_123）
3. 当第一个用户发布流时，SRS自动创建流
```

**解散房间**（业务层实现）:
```
1. 业务服务器决定解散房间
2. 调用 DELETE /api/v1/clients/{client_id} 踢掉所有用户
3. 或等待所有用户自然离开
4. 当没有用户时，SRS自动清理流
5. 业务服务器删除数据库中的房间记录
```

## 接口分类总结

| 分类 | 接口数量 | 是否必需 | 说明 |
|------|---------|---------|------|
| **核心业务接口** | 4 | ✅ 必须 | RTC播放、发布、WHIP、WHEP |
| **房间管理接口** | 3 | ⭐ 推荐 | 查询流、查询用户、踢用户 |
| **系统监控接口** | 2 | ⚪ 可选 | 系统状态、Prometheus指标 |

**总计需要对外暴露**: **9个接口**（4个核心 + 3个管理 + 2个监控）

> **注意**: SRS不提供创建/解散房间接口，房间管理由业务层实现。详见"关于房间管理的重要说明"章节。

---

## 安全建议

1. **接口访问控制**: 
   - 建议通过Nginx等反向代理添加IP白名单
   - 或使用API网关进行鉴权

2. **敏感接口保护**:
   - `/api/v1/clients/` DELETE接口需要业务层鉴权
   - 不要直接暴露给前端

3. **HTTPS使用**:
   - 生产环境建议使用HTTPS
   - 保护WebRTC SDP交换过程

4. **限流保护**:
   - 对RTC接口进行限流，防止恶意请求
   - 对查询接口进行限流，防止过度查询

## 示例代码

### Node.js调用示例

```javascript
// 用户加入房间
async function joinRoom(roomId, sdpOffer) {
  const response = await fetch('http://srs-server:1985/rtc/v1/play/', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      sdp: sdpOffer,
      streamurl: `webrtc://your-domain.com/live/${roomId}`,
      api: 'http://your-api-server/api/callback'
    })
  });
  
  const data = await response.json();
  return data.sdp; // SDP answer
}

// 查询房间信息
async function getRoomInfo(roomId) {
  const response = await fetch(`http://srs-server:1985/api/v1/streams/${roomId}`);
  const data = await response.json();
  return data.stream;
}

// 解散房间（踢掉所有用户）
async function dismissRoom(roomId) {
  // 1. 查询房间内所有用户
  const response = await fetch(`http://srs-server:1985/api/v1/streams/${roomId}`);
  const streamData = await response.json();
  
  // 2. 获取所有客户端ID（需要根据实际响应结构调整）
  const clients = await fetch(`http://srs-server:1985/api/v1/clients/?start=0&count=100`);
  const clientsData = await clients.json();
  
  // 3. 过滤出该房间的用户并踢掉
  const roomClients = clientsData.clients.filter(c => c.stream === roomId);
  for (const client of roomClients) {
    await fetch(`http://srs-server:1985/api/v1/clients/${client.id}`, {
      method: 'DELETE'
    });
  }
  
  // 4. 业务层删除房间记录（这里只是示例，实际应该在业务服务器中实现）
  // await deleteRoomFromDatabase(roomId);
}
```

## 注意事项

1. **WebRTC SDP格式**: RTC接口的SDP需要是完整的WebRTC SDP格式
2. **流URL格式**: `webrtc://domain/app/stream_name`
3. **会话管理**: 保存sessionid和token，用于后续断开连接
4. **错误处理**: 所有接口返回`code != 0`时表示失败
5. **超时设置**: RTC接口建议设置30秒超时
6. **重试机制**: 网络错误时建议重试，但要注意幂等性
